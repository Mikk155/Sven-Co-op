Allows the server to precache multiple assets in two different ways

Open the json configuration file and you will see two objects where you can add assets to precache 


{
    "AlwaysPrecached": { },
    "DynamicPrecache": { },
    "MaxDynamicPrecache": 5
}

"AlwaysPrecached" as it's name suggest will always precache the assets inside it.

While "DynamicPrecache" will precache a set number of assets between different maps ("MaxDynamicPrecache")

The keys in the object represents the asset to precache and the value is a number representing the asset type 


"AlwaysPrecached":
{
    "models/player/Hololive_Fubuki/Hololive_Fubuki.mdl": 2,
    "models/player/Hololive_Fubuki/Hololive_Fubuki.bmp": 1,
    "oghs/chatsounds/alo.wav.wav": 3,
    "point_checkpoint": 4
}

So if you want to precache a mdl or spr file you set the value to "2" corresponding to the next table: 


enum PrecacheType
{
    Dictionary = 0,
    Generic = 1,
    Model = 2,
    Sound = 3,
    Entity = 4
};

The value could also be an object, in wich case that will be a entity to precache with custom key values. so you can precache custom model lists, sound lists etc. 


"AlwaysPrecached":
{
    "monster_houndeye":
    {
        "model": "models/hlclassic/houndeye.mdl"
    }
}
