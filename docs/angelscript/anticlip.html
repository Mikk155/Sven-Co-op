Allow players to pass through other players and allied monsters just like you'd do in Left 4 Dead

Anti-clip unlike Semi-clip doesn't resort to do some hacking on the players but completelly overwrite the collision entity list in the player movement code (Server-side)

This "Anti Clipping" is not limited to players. There is a json config "npc_clip" which when set to false players can pass through allied monsters as well.

With the json config "projectiles_clip" this will also affect your projectiles. i.e rockets, spores, grenades, bolts, squeaks etc. the same with custom weapons as long as their projectiles defines the player as their owner.

Players can still "boost" onto other players or monsters if the json variable "player_boost" is true.

We previously said "Server-Side" so all of that would actually feel really laggy to the clients but there's a label in the json config called "client_prediction"
When the "active" option is set to true the plugin will register the AddToFullPack hook. this is quite expensive so we made it to be an option for you instead.
But enabling it is not an absolute risk. We've made a system for that to know when the server is experiencing slow frame rates and dynamically remove / register that hook.

[Anti-Clip] The Server is experiencing slow frame rates. Disabling client prediction...
...
[Anti-Clip] Server recovered. Re-enabling prediction.

The client prediction section also has a rendering setting for rendermode & renderamt so whatever you're colliding with an entity it will use that render settings.


This sections is entirelly cosmetic. the most relevant use of this is to tell the client that the entity we're hiting is not solid.
Otherwise the client things it is solid and so it kicks back them for a moment. but in reality at the server side we're not colliding at all.
The main purpose is just to give clients a bit of sync.
There's the possibility for clients to use the client command "cl_solid_players 0" to avoid using AddToFullPack but we've experienced some issues while clients are on spectator mode.
